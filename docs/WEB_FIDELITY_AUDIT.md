## 已核实的真实交互

### 英雄

- 英雄等级使用 1–50 选择器；页面不提供独立“英雄 Rank”输入。
- 种子数量为 0–80 的单一选择器，线上写入生命、攻击、防御三个种子字段。
- 收藏卡牌为 0–3；对应攻防血增益为 0%、5%、10%、25%。
- 自带技能由职业 `innate` 自动决定且不可替换。
- 最多检查四个可选槽；`sklNLv=0` 表示槽位不存在，否则在对应英雄等级解锁。骑士为 5、10、23 级三个槽。
- 技能可选范围按 `classes` 中的职业 ID、职业类型或通配符筛选；同一技能族不能重复占用多个槽。
- 技能卡显示当前一阶效果和“满级技能效果”。英雄面板上的技能等级不是手工输入，而是由装备元素值、技能族元素门槛和职业最高技能阶动态得出。基础职业最高 3 阶，晋升职业最高 4 阶。
- “泰坦之塔/墓”是属性预览开关；墓生灵精萃的本装备全属性倍率在该场景翻倍。

### 勇士

- 勇士拥有独立数据模型：固定团队技能、勇士阶数、勇士之魂和两个装备位；没有英雄可选技能槽。
- 勇士等级为 1–50。
- 勇士阶数为 1–11，再继续到 11+1…11+60；内部可无损表示为 1–71。
- 种子和收藏卡牌的选择范围与英雄相同。
- 勇士之魂是布尔开关，会进入勇士阶数属性公式；它不是“泰坦之塔/墓”的同义词。
- 阿尔贡 11 阶显示固定团队技能“圣骑光环”4 级；团队技能等级由勇士阶数升级节点决定。
- 勇士有使魔和光环之歌两个装备槽；每个槽仍使用装备品质、星能铸造、超越、元素附魔和精萃附魔规则。

### 装备

- 六个英雄槽由职业的 `slot1..slot6` 类型约束；目录按装备阶数从高到低展示。
- 装备品质选项是普通、高级、无暇、史诗、传奇。
- 星能铸造是开/关；超越也是开/关，不是 0–5 等级输入。
- 元素附魔和精萃附魔是两个独立目录，并非一个二选一的“符文”字段。
- 在线选择流程允许先选装备，再选两种附魔，状态立即反映在卡片和实时属性中；关闭选择器不会清空已选状态。

### 排序

- 英雄阵容提供职业排序和元素排序两个互斥模式。
- 装备目录按阶数降序，同阶按在线数据顺序/名称稳定展示。
- 技能目录先按稀有度从高到低，再按技能数据顺序稳定展示；每个技能族只展示一阶卡片。

### 冒险任务

- 新增任务分组时，线上会同时创建一个默认任务；单个体系最多显示 48 个任务。
- 任务目录分为普通冒险、黄金城、泰坦塔、快闪四类。普通冒险先选地图/首领图，再选简单、中等、困难、究极；黄金城先选普通图/首领图，再选难度 1–13。
- 泰坦塔不再经过二级难度页：目录直接列出第 1 层至第 30 层和泰坦之墓，点击层数即完成选择。
- 冒险强化道具有无、威力、超级威力、特级威力四档。三档加成分别为攻防 +20%/暴击 +10%，攻防 +40%/暴击 +15%，攻防 +80%/暴击 +30%/暴伤 +50%。
- 精英怪不是布尔值，而是无、敏捷、巨大、凶残、富有、传奇六种互斥类型；选择值会持久化，并在配置卡与结果详情中显示具体类型。
- 元素屏障由 `simulationConfig.selectedElement` 选择：未设置表示自动，具体元素覆盖自动选择，`force` 表示无屏障。可选元素取自任务数据；它不是六个由用户手填的屏障数值。

## “星图/技能树”边界

目标页的 `HeroLineup-DP_2OddU.js` 不包含 `星图`、`技能树`、`skillTreeNodes` 或 `skillTreePoints` 引用。仓库中的 `skillTreeNodes.json` 与 `skillTreePoints.json` 是游戏全局天赋/商店数据，不能在没有在线入口证据时冒充 hero-lineup 的英雄星图功能。因此当前状态为：**不属于已核实的目标页功能，未伪造实现**。若后续确认“星图”指另一个线上页面，应单独提供入口并建立新的交互审计。
