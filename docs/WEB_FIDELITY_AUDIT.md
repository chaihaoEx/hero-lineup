## 已核实的真实交互

### 英雄

- 英雄等级使用 1–50 选择器；页面不提供独立“英雄 Rank”输入。
- 种子数量为 0–80 的单一选择器，线上写入生命、攻击、防御三个种子字段。
- 收藏卡牌为 0–3；对应攻防血增益为 0%、5%、10%、25%。
- 自带技能由职业 `innate` 自动决定且不可替换。
- 最多检查四个可选槽；`sklNLv=0` 表示槽位不存在，否则在对应英雄等级解锁。骑士为 5、10、23 级三个槽。
- 技能可选范围按 `classes` 中的职业 ID、职业类型或通配符筛选；同一技能族不能重复占用多个槽。
- 技能卡显示当前一阶效果和“满级技能效果”。英雄面板上的技能等级不是手工输入，而是由装备元素值、技能族元素门槛和职业最高技能阶动态得出。基础职业最高 3 阶，晋升职业最高 4 阶。
- “泰坦之塔/墓”是属性预览开关；墓生灵精萃的本装备全属性倍率在该场景翻倍。

### 勇士

- 勇士拥有独立数据模型：固定团队技能、勇士阶数、勇士之魂和两个装备位；没有英雄可选技能槽。
- 勇士等级为 1–50。
- 勇士阶数为 1–11，再继续到 11+1…11+60；内部可无损表示为 1–71。
- 种子和收藏卡牌的选择范围与英雄相同。
- 勇士之魂是布尔开关，会进入勇士阶数属性公式；它不是“泰坦之塔/墓”的同义词。
- 阿尔贡 11 阶显示固定团队技能“圣骑光环”4 级；团队技能等级由勇士阶数升级节点决定。
- 勇士有使魔和光环之歌两个装备槽；每个槽仍使用装备品质、星能铸造、超越、元素附魔和精萃附魔规则。

### 装备

- 六个英雄槽由职业的 `slot1..slot6` 类型约束；目录按装备阶数从高到低展示。
- 装备品质选项是普通、高级、无暇、史诗、传奇。
- 星能铸造是开/关；超越也是开/关，不是 0–5 等级输入。
- 元素附魔和精萃附魔是两个独立目录，并非一个二选一的“符文”字段。
- 在线选择流程允许先选装备，再选两种附魔，状态立即反映在卡片和实时属性中；关闭选择器不会清空已选状态。

### 排序

- 英雄阵容提供职业排序和元素排序两个互斥模式。
- 默认职业排序按 `classes.json` 目录顺序，同职业按英雄名 `localeCompare`；元素排序固定为光、暗、火、水、土、风、全元素，再沿用职业与英雄名两层顺序。
- 勇士和英雄主阵容图标都叠加左上角元素图片徽章；全元素职业使用 `all` 徽章，卡片不额外显示元素文字。
- 装备目录按阶数降序，同阶按解锁等级降序，再保持 `items.json` 原始顺序；英雄等级低于装备可用阶数时不展示该装备。
- 技能目录先按稀有度从高到低，同稀有度按 `skills.json` 原始逆序展示；每个技能族只展示一阶卡片。Rust 目录层显式透传 `sourceOrder`，避免本地化名称排序破坏线上次序。
- 装备槽按实际 `elementAffinity` / `spiritAffinity` 显示元素亲和和精萃亲和；亲和附件的 1.5 倍规则仍由 Rust 数据引擎计算。

### 冒险任务

- 新增任务分组时，线上会同时创建一个默认任务；单个体系最多显示 48 个任务。
- 任务目录分为普通冒险、黄金城、泰坦塔、快闪四类。普通冒险先选地图/首领图，再选简单、中等、困难、究极；黄金城先选普通图/首领图，再选难度 1–13。
- 泰坦塔同样是两级选择：先选第 1 层至第 30 层或泰坦之墓；普通楼层再选阿尔法、贝塔、伽马、德尔塔、艾普斯龙、奇异，泰坦之墓再选测试墓穴、伽马墓、德尔塔墓穴、伊普西隆墓、恐怖墓穴。难度卡由泰坦背景图和变体图标叠加组成。
- 快闪目录按线上固定顺序展示 17 张地图，点击地图后再进入简单、中等、困难、究极四难度页；中文名称使用 `fq_*_name` 文本键，不能用英文 family ID 兜底充当界面文案。
- 冒险强化道具有无、威力、超级威力、特级威力四档。三档加成分别为攻防 +20%/暴击 +10%，攻防 +40%/暴击 +15%，攻防 +80%/暴击 +30%/暴伤 +50%。
- 精英怪不是布尔值，而是无、敏捷、巨大、凶残、富有、传奇六种互斥类型；仅普通非 Boss 冒险显示该控件。黄金城、泰坦塔、快闪没有精英/屏障或额外“泰坦塔”复选框，但满足数据条件时仍会出现经验强化、泰坦词条与墓穴楼层。
- 元素屏障由 `simulationConfig.selectedElement` 选择：未设置表示自动，具体元素覆盖自动选择，`force` 表示无屏障。仅普通冒险且任务有候选屏障时显示；切换地图保留强化道具，但重置精英怪、屏障和旧模拟结果。
- 每个任务最多上阵一名勇士；已有勇士时“选择成员添加到任务”只继续列出英雄。离线目录、拖放、旧数据归一化和两种导入校验共同执行该约束。
- 成员目录默认排除同一分组其他任务已上阵的成员，而不是全体系过滤；切换“全部成员”后把偏好写入 `heroLineup_taskMemberPickerAllMembers`。离线端已采用相同范围与持久化键。
- 成员弹窗上限 672px，成员网格在小屏/1024/1280 分别为 4/5/6 列；成员目录及任务卡头像左上角均使用真实元素图片角标，任务卡为 18px，目录按断点为 16/20/24px。
- 点击“测试冒险”不打开参数弹窗，而是直接运行固定 10,000 次并覆盖卡片结果；卡片结果只显示三位小数成功率、“查看详情”和“测试冒险”。
- “查看详情”弹窗展示五项任务汇总，并分别展示每位成员的存活率、平均伤害和平均剩余生命。离线 Rust 命令层现已透传成员级结果，避免用队伍汇总值伪装个人统计。
- 详情图片自动以 2× 像素比生成；准备失败、复制成功/失败文案与线上归档一致，移动端与线上相同只保留下载图片按钮。

## “星图/技能树”边界

目标页的 `HeroLineup-DP_2OddU.js` 不包含 `星图`、`技能树`、`skillTreeNodes` 或 `skillTreePoints` 引用。仓库中的 `skillTreeNodes.json` 与 `skillTreePoints.json` 是游戏全局天赋/商店数据，不能在没有在线入口证据时冒充 hero-lineup 的英雄星图功能。因此当前状态为：**不属于已核实的目标页功能，未伪造实现**。若后续确认“星图”指另一个线上页面，应单独提供入口并建立新的交互审计。

### 装备需求统计

- 归档组件 `GB` 接收当前体系全部英雄或全部勇士 ID，不以任务上阵状态过滤。
- 主装备、元素附魔、精萃附魔分别以 `category_itemId_quality` 聚合；两种附魔沿用所在装备槽的品质。
- 卡片固定按主装备、元素附魔、精萃附魔排序；同类再按传奇至普通、阶数降序、中文名称排序。
- “已有”数量以 `itemId_quality` 为键，允许输入暂时为空；非空输入向下取整且不小于零。需要数量在已有大于零时划线，并显示红色剩余或绿色零。
- 弹窗开启时复制体系库存，关闭时一次性回写；离线端将该字段纳入体系 SQLite、跨平台体系文件和完整备份。桌面固定四列，窄于 768px 时两列。

## 本轮验收结论

已完成勇士团队技能和完整装备状态落库、技能等级边界测试、职业/元素/装备/技能排序 E2E、多尺寸截图、断网审计、macOS `.app`/`.dmg` 签名与干净目录启动验证，并保留 Windows 兼容的 Tauri/Rust/React 单源码结构。当前明确不声称完全一致的是部分战斗模拟公式：特殊神器/职业修正、精英附加 AOE、部分任务 qmodifier 组合仍缺少足够在线输入输出样本；其边界逐项记录在 `KNOWN_DIFFERENCES.md`，不以猜测值伪装精确还原。
