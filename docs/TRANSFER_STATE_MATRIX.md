# 本地传输与数据更新状态迁移

线上页面的体系口令依赖服务端索引；`.zyslineup`、`.zysbackup` 和 `.zysdata`
是离线桌面版新增的本地能力，没有可逐像素照搬的线上文件对话框。本表约束这些入口在
macOS/Windows 共用 React 与 Rust 命令层中的可观察状态，避免取消、失败或版本冲突误报成功。

| 入口 | 初始操作 | 用户取消 | 成功 | 失败/冲突 | 数据不变量 |
|---|---|---|---|---|---|
| 导出体系 | Rust 先生成带 checksum 的单体系 envelope，再打开保存对话框 | `pick_write_interchange=false`；不显示成功提示 | 写入后提示“体系已导出为跨平台文件” | 显示 Rust 原始可行动错误；不再产生未处理 Promise rejection | 不修改体系、dirty 状态或 SQLite |
| 导入体系 | 打开 `.zyslineup` 对话框，读取后校验 schema、checksum 和游戏数据版本 | `pick_read_interchange=null`；不显示提示、不增加卡片 | Rust 持久化后追加体系、切换到首个导入体系并提示数量 | 显示校验/版本错误；当前体系和卡片数量不变 | 只有完整校验及 SQLite 写入成功后更新前端状态 |
| 完整备份 | Rust 读取体系、模板和设置并生成 `.zysbackup` | 保存对话框取消时不提示成功 | 文件写入后提示“完整备份已写入本机” | 显示读取、编码或写入错误 | 不修改用户数据库 |
| 恢复备份 | 选择文件后再显示替换确认 | 任一对话框取消均无变化 | Rust 单事务替换后刷新体系并提示事务恢复 | checksum、版本、引用或事务错误直接显示；旧数据库保留 | 前端只接收 Rust 已提交后的完整体系列表 |
| 更新本地数据 | 有 dirty 修改时先询问是否保存，再选择 `.zysdata` | 保存确认或文件选择取消均停止，且不提示成功 | 完整验包、应用级预检、原子切换后重载目录并报告版本/过期模拟数 | 显示验证或安装错误；原数据目录和用户库不变 | 安装锁串行化；任何失败均不得切换活动内容根 |

## 自动化证据

- `e2e/native-contract.spec.ts` 覆盖体系导出取消、写入失败、体系导入取消、版本冲突、
  数据包选择取消和验包失败；分别断言无误报、错误提示和体系卡数量不变。
- `hero-domain` 覆盖 schema/checksum/游戏数据版本不兼容；`hero-storage` 覆盖备份事务恢复。
- Tauri 内容管理测试覆盖 ZIP 截断、哈希篡改、最低应用版本、路径逃逸、预检失败和原子切换回滚。
